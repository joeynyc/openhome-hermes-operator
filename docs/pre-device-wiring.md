# Pre-Device Wiring Runbook

This runbook lets us wire and test almost everything before the OpenHome DevKit arrives.

## 1. Install local dev dependencies

```bash
cd /home/zerocool/projects/openhome-hermes-operator
python3 -m venv .venv
source .venv/bin/activate
pip install -e '.[dev]'
```

## 2. Test bridge in fake mode

Terminal 1:

```bash
source .venv/bin/activate
./scripts/run_bridge_fake.sh
```

Terminal 2:

```bash
export HERMES_OPERATOR_TOKEN=dev-token
./scripts/demo_curl.sh "check my local model server"
python3 -m hermes_operator.cli "check my local model server"
```

Expected spoken summary:

```text
Hermes fake mode is working. I received: check my local model server
```

This proves:

- Bridge starts.
- Bearer token auth works.
- OpenHome-style task payload works.
- Voice-safe response shape works.
- Local simulator works.

## 3. Enable Hermes API Server

Hermes API Server is what the bridge calls in live mode.

Add to `~/.hermes/.env`:

```bash
API_SERVER_ENABLED=true
API_SERVER_KEY=dev-token
API_SERVER_PORT=8642
API_SERVER_HOST=127.0.0.1
API_SERVER_MODEL_NAME=hermes-agent
```

Restart gateway if using gateway mode:

```bash
hermes gateway restart
```

Or run whichever Hermes API Server workflow is configured locally.

Health check:

```bash
curl -s http://127.0.0.1:8642/health
```

Model check:

```bash
curl -s http://127.0.0.1:8642/v1/models \
  -H "Authorization: Bearer $API_SERVER_KEY"
```

## 4. Test bridge against live Hermes

Terminal 1:

```bash
source .venv/bin/activate
export HERMES_API_KEY=dev-token
export HERMES_OPERATOR_TOKEN=dev-token
./scripts/run_bridge_live.sh
```

Terminal 2:

```bash
export HERMES_OPERATOR_TOKEN=dev-token
python3 -m hermes_operator.cli "Say hello in one short sentence."
```

Expected:

A concise spoken sentence generated by Hermes.

## 5. LAN settings for the eventual OpenHome device

The OpenHome device will need to reach the bridge on the LAN.

On DGX Spark use LAN IP:

```text
http://192.168.1.201:8787/run
```

Set ability env/config:

```bash
OPENHOME_HERMES_BRIDGE_URL=http://192.168.1.201:8787/run
OPENHOME_HERMES_BRIDGE_TOKEN=dev-token
OPENHOME_HERMES_TIMEOUT=240
```

Keep Hermes API Server private on localhost. Only expose the small bridge to LAN.

## 6. Package ability zip

```bash
cd /home/zerocool/projects/openhome-hermes-operator

```

Upload that zip to the OpenHome dashboard when the device/account is available.

## 7. Final pre-device validation

Run this before considering the repo ready for the physical OpenHome device:

```bash
cd /home/zerocool/projects/openhome-hermes-operator
source .venv/bin/activate
pip install -e '.[dev]'
python -m pytest tests -v
python -m build
./scripts/package_ability.sh
```

Expected local state:

- tests pass
- `dist/` contains the Python package build outputs
- `/tmp/hermes-operator-openhome-ability.zip` contains `hermes-operator/main.py` and `hermes-operator/README.md`
- `git status --short` is clean after committed docs/code changes
